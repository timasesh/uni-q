СТРУКТУРА ПРОЕКТА uni-q
=======================

Общее назначение проекта
------------------------
`uni-q` — это fullstack-приложение для электронной очереди в университете.
В проекте есть:
- frontend на React + Vite;
- backend на Express + Socket.IO;
- локальная база SQLite;
- дополнительные модули для PostgreSQL / Supabase;
- административный, менеджерский и студенческий интерфейсы.


1. Корень проекта
-----------------

Папки в корне:
- `src/` — весь клиентский React-код.
- `server/` — вспомогательные серверные модули для PostgreSQL / Supabase.
- `public/` — статические файлы и изображения.
- `scripts/` — служебные одноразовые скрипты.
- `supabase/` — SQL-миграции для PostgreSQL.

Файлы в корне:

- `server.ts`
  Главный backend-файл проекта. Поднимает Express-сервер, Socket.IO, подключает SQLite, создаёт таблицы, сиды, API для студентов, менеджеров и админов, а также синхронизацию с PostgreSQL при необходимости.

- `package.json`
  Основной файл npm-проекта. Содержит зависимости, dev-зависимости и команды запуска (`dev`, `build`, `start`).

- `package-lock.json`
  Lock-файл npm. Нужен для точного воспроизводимого набора зависимостей.

- `README.md`
  Краткая документация по проекту: запуск, адреса, тестовые аккаунты, базовые инструкции.

- `DEPLOY-RENDER.md`
  Подробная инструкция по деплою проекта на Render и работе с Supabase / PostgreSQL.

- `Dockerfile`
  Описание сборки Docker-образа для продакшн-запуска приложения.

- `render.yaml`
  Конфигурация Render Blueprint для автоматического развёртывания.

- `index.html`
  HTML-шаблон, через который Vite подключает React-приложение.

- `vite.config.ts`
  Конфигурация Vite: React-плагин, dev-server, прокси на backend.

- `tsconfig.json`
  Конфигурация TypeScript для проекта.

- `tailwind.config.js`
  Конфигурация Tailwind CSS: какие файлы сканировать и как применять стили.

- `postcss.config.js`
  Конфигурация PostCSS для Tailwind и автопрефиксов.

- `.env.example`
  Пример переменных окружения: путь к SQLite, параметры Microsoft OAuth, PostgreSQL и прочее.

- `.gitignore`
  Список файлов и папок, которые Git не должен отслеживать.

- `.dockerignore`
  Список файлов, которые не должны попадать в Docker context.

- `uni-q.sqlite`
  Локальная база данных SQLite. Хранит данные приложения. Это не исходный код, а рабочая база.

- `ОТВЕТЫ_ПО_ЗАДАЧАМ.txt`
  Внутренний текстовый файл с заметками/ответами по задачам проекта.


2. Папка src/
-------------
`src/` — основная клиентская часть проекта на React.

Файлы верхнего уровня:

- `src/main.tsx`
  Точка входа frontend-приложения. Монтирует React-приложение в DOM и подключает провайдеры/роутинг.

- `src/App.tsx`
  Главный компонент маршрутизации приложения. Определяет переходы между студенческой, менеджерской и админской частями.

- `src/api.ts`
  Вспомогательные функции для запросов к backend API.

- `src/i18n.ts`
  Система локализации интерфейса. Содержит словари переводов и логику выбора языка.

- `src/types.ts`
  Общие TypeScript-типы для frontend-части.

- `src/schools.ts`
  Справочные данные по школам, специальностям, языкам и связанным сущностям.

- `src/style.css`
  Глобальные стили приложения.


3. Папка src/admin/
-------------------
- `src/admin/AdminApp.tsx`
  Основной интерфейс администратора. Здесь сосредоточены админские экраны: логин, управление сотрудниками, окнами, отчётами и статистикой.


4. Папка src/components/
------------------------
- `src/components/ChatWidget.tsx`
  Виджет чата/помощи для пользователя.

- `src/components/ManagerWorkTimer.tsx`
  Компонент таймера рабочего времени менеджера.

- `src/components/SchemeImage.tsx`
  Компонент для отображения схемы/изображения с запасным форматом и безопасной загрузкой.


5. Папка src/context/
---------------------
- `src/context/AdminContext.tsx`
  React Context для состояния администратора: авторизация, профиль, доступ к админской части.

- `src/context/ManagerContext.tsx`
  React Context для состояния менеджера: идентификатор, сессия, связанное состояние интерфейса.


6. Папка src/data/
------------------
- `src/data/faq.ts`
  Источник данных для FAQ-страницы. Хранит вопросы и ответы.


7. Папка src/lib/
-----------------
- `src/lib/advisorScope.ts`
  Логика определения области приёма менеджера: какие студенты/направления к нему относятся.

- `src/lib/advisorWorkSync.ts`
  Логика синхронизации рабочего времени менеджера между клиентом и сервером.

- `src/lib/backendDateTime.ts`
  Вспомогательные функции для корректной обработки времени и дат, приходящих с backend.

- `src/lib/bookingCallable.ts`
  Проверяет, можно ли уже вызывать студента по времени брони.

- `src/lib/bookingSlots.ts`
  Генерация и обработка временных слотов для записи/бронирования.

- `src/lib/brand.tsx`
  Компоненты и константы, связанные с брендингом и логотипом.

- `src/lib/cn.ts`
  Небольшой helper для объединения CSS-классов.

- `src/lib/deskWindow.ts`
  Логика, связанная с окнами/столами/номерами приёма.

- `src/lib/formatAdvisorReceptionSummary.ts`
  Формирует человекочитаемое описание настроек приёма менеджера.


8. Папка src/pages/
-------------------
- `src/pages/AdvisorPage.tsx`
  Главная страница менеджера. Показывает очередь, позволяет вызывать студентов, завершать приём, смотреть историю.

- `src/pages/AdvisorSettingsPage.tsx`
  Страница настроек менеджера: зона приёма, пароль, доступные школы/курсы/языки и другие параметры.

- `src/pages/FaqPage.tsx`
  Страница FAQ для студентов.

- `src/pages/StudentEntryPage.tsx`
  Начальная страница входа студента перед регистрацией в очередь.

- `src/pages/StudentPage.tsx`
  Основная страница студента: регистрация в очередь, просмотр статуса талона, бронь времени, отзыв и другое.


9. Папка server/
----------------
Папка `server/` содержит вспомогательные backend-модули, которые расширяют основной `server.ts`.

- `server/pgCore.ts`
  Модуль для работы с PostgreSQL как с дополнительным хранилищем/зеркалом основных данных и отчётности.

- `server/pgHistory.ts`
  Модуль для работы с историей визитов в PostgreSQL.


10. Папка scripts/
------------------
- `scripts/create-admin.ts`
  Служебный скрипт для ручного создания администратора в SQLite-базе с хэшированием пароля.


11. Папка supabase/
-------------------
Папка для SQL-миграций PostgreSQL / Supabase.

11.1. Папка `supabase/migrations/`

- `supabase/migrations/20260412120000_initial.sql`
  Начальная SQL-миграция. Создаёт базовые таблицы PostgreSQL, аналогичные структуре SQLite.

- `supabase/migrations/20260425210000_study_years_scope.sql`
  Дополнительная миграция, связанная с областями приёма и учебными годами.

13. Архитектура проекта простыми словами
----------------------------------------
- `src/` отвечает за интерфейс и пользовательскую логику.
- `server.ts` отвечает за сервер, API, авторизацию, очереди, базу и реальное поведение системы.
- `server/` дополняет сервер интеграцией с PostgreSQL.
- `public/` хранит статические изображения и ресурсы.
- `scripts/` нужен для разовых технических действий.
- `supabase/` нужен для SQL-структуры PostgreSQL и миграций.


14. Что где используется
------------------------
- Студент работает в основном через `src/pages/StudentEntryPage.tsx`, `src/pages/StudentPage.tsx`, `src/pages/FaqPage.tsx`.
- Менеджер работает через `src/pages/AdvisorPage.tsx` и `src/pages/AdvisorSettingsPage.tsx`.
- Администратор работает через `src/admin/AdminApp.tsx`.
- Все они обращаются к backend API, который реализован в `server.ts`.
- История и отчёты могут дополнительно использовать PostgreSQL через `server/pgCore.ts` и `server/pgHistory.ts`.


15. Важные замечания
--------------------
- `server.ts` — самый главный и самый крупный файл backend-логики.
- `src/admin/AdminApp.tsx`, `src/pages/AdvisorPage.tsx` и `src/pages/StudentPage.tsx` — ключевые большие файлы интерфейса.
- `uni-q.sqlite` — это живая база данных, её нельзя рассматривать как обычный кодовый файл.
- Конфигурационные файлы (`vite.config.ts`, `tsconfig.json`, `tailwind.config.js`, `postcss.config.js`) не содержат бизнес-логику, но нужны для сборки и запуска проекта.


16. Краткий итог
----------------
Проект устроен как единый fullstack-репозиторий:
- frontend лежит в `src/`;
- backend сосредоточен в `server.ts`;
- PostgreSQL-интеграция вынесена в `server/`;
- статические ресурсы лежат в `public/`;
- служебные скрипты лежат в `scripts/`;
- SQL-миграции лежат в `supabase/migrations/`.
